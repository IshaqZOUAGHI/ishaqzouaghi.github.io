Vous trouverez dans ce billet une sélection de liens pratiques autour des technologies Java et autres qui m'ont particulièrement intéressées cette semaine 07 (février 2016). J'en profite pour vous souhaiter de très bonnes fêtes de fin d'année.

Java

* La version [GWT 2.8.0 (Beta 1)](http://www.gwtproject.org/release-notes.html#Release_Notes_2_8_0_BETA1) est disponible en téléchargement. Au menu, le support de Java 8 et JsInterop pour facilement mapper avec des bibliothèques JS existantes.

Eclipse

* La [fondation Eclipse](https://mmilinkov.wordpress.com/2016/01/15/investing-in-eclipse-tools/) recrute (ou l'a peut-être déjà fait) une personne en charge d'améliorer l'environnement de développement. C'est plutôt une bonne nouvelle.
* [Eclipse CHE](https://www.eclipse.org/community/eclipse_newsletter/2016/january/article1.php), c'est l'environnement de développement intégré dans le navigateur. Il y a quelques années quand on parlait d'Eclipse 4, il se murmurait l'envie d'avoir ce type d'évolution. Je pense que c'est bien parti pour être réalisé. Il faut juste que l'éditeur dans le navigateur soit de même qualité voir meilleur que ce qui est proposé par son homologue bureau ou voir IntelliJ.

Divers

* [Markdown preview](https://github.com/revolunet/sublimetext-markdown-preview) est un plugin pour Sublimetext afin de visualiser un document Markdown.
* [Asciinema](https://asciinema.org/) est un outil pour enregistrer le contenu de votre console (Linux et MacOS X). Vous pouvez enregistrer sur la plateforme en ligne ou vers un fichier JSON.
* [Asciinema2GIF](https://github.com/tav/asciinema2gif) est util, pour transformer un contenu Asciinema en GIF.
* [ttystudio](https://github.com/chjj/ttystudio) est un autre outil pour sauvegarder son terminal.
* [GitUp](http://gitup.co/) un outil pour vous aider à gérer le contenu d'un GIT.
* Très [bonne explication](https://www.ctl.io/developers/blog/post/dockerfile-entrypoint-vs-cmd/) de la différence entre CMD et ENTRYPOINT au niveau de Docker.
* [Wekan](https://wekan.io/) c'est le [Trello](https://trello.com/) open source.
* [Electron](http://electron.atom.io/) permet de transformer une application HTML/JS/Node.js en une application bureau.
* Quelques outils pour manipuler du PDF et se passer de cette usine à gaz payante d'Acrobat Writer. [Briss](http://briss.sourceforge.net/) pour rogner (crop) une page PDF. [PDFShuffler](https://sourceforge.net/projects/pdfshuffler/) pour ajouter des pages, rogner, faire des rotations, supprimer des pages... [PDFTk](https://www.pdflabs.com/tools/pdftk-the-pdf-toolkit/) fait aussi beaucoup de choses. Merci à Henri pour les tuyaux.
* Une [plateforme](http://johnny-five.io/) qui permet de faire fonctionner du Node.js sur pratiquement toutes les cartes micro-controlleurs. Le site web est très bien fait.
